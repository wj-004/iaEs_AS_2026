# Alcance vs Límites en Ingeniería de Software

Basado en Ingeniería del Software: Un enfoque práctico – Pressman

---

# 📌 1. ALCANCE (Scope)

El alcance define qué hace el sistema.

## ✔ Definición
El alcance describe las funcionalidades, procesos y responsabilidades que el sistema incluye.

## 📌 Pregunta clave
¿Qué hace el sistema?

## 🟦 Características del alcance
- Es funcional
- Describe capacidades del sistema
- Se expresa en términos positivos

## 📦 Ejemplos
- Gestionar insumos
- Registrar ventas
- Controlar stock
- Administrar producción

👉 El alcance es lo que el sistema SÍ hace

---

# 📌 2. LÍMITES (Boundaries / Constraints)

Los límites definen hasta dónde llega el sistema y bajo qué condiciones opera.

## ✔ Definición
Los límites son restricciones que determinan el contexto del sistema y lo que queda fuera de su responsabilidad.

## 📌 Pregunta clave
¿Hasta dónde llega el sistema y en qué condiciones funciona?

## 🟥 Características de los límites
- Son restrictivos
- Definen el contexto del sistema
- Pueden ser técnicos, funcionales u organizacionales

## 📦 Ejemplos
- Sistema para una sola empresa
- Sin sucursales
- Sin múltiples usuarios o roles
- Solo aplicación web
- Sin integración externa
- Sin gestión de facturación

👉 Los límites definen el borde del sistema

---

# 📌 3. DIFERENCIA CLAVE

| Concepto | Pregunta | Tipo |
|----------|----------|------|
| Alcance | ¿Qué hace el sistema? | Funcional |
| Límites | ¿Hasta dónde llega el sistema? | Contextual / restrictivo |

---

# 📌 4. MODELO MENTAL

Piensa en el sistema como un círculo:

- 🟦 Alcance = lo que está dentro del círculo (funciones)
- 🟥 Límites = el borde del círculo (restricciones)

---

# 📌 5. CONCLUSIÓN

- El alcance define funcionalidades del sistema
- Los límites definen el contexto y restricciones
- No son opuestos, sino complementarios
- Ambos son necesarios para definir correctamente un sistema

---

✔ Un sistema bien definido siempre separa claramente alcance y límites
